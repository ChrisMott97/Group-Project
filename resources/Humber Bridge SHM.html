3.4 Setting up shmYm

There is a MATLAB class shmYm developed by Ki to enable a user to download SHM data very easily to MATLAB Workspace with two lines of MATLAB commands. When a user issues the commands in MATLAB command window, shmYm connects to the VESHM MySQL Database in Univ. of Exeter, downloads and saves the data into yours MATLAB Workspace. At this moment, you probably have a Dropbox invitation from Ki (If not, please ask Ki). To set up shmYm, you need to do followings.

  1. For Windows 7 or XP, install following VC++ Runtime component. Nothing is required for Linux and OSX.
    1. Windows 7/8/10
    2. Windows XP: x86 version, x64 version
  2. Add Dropbox folder "Dropbox/EPSRC_SHM/01-Library" into your MATLAB Path.
  3. Restart your MATLAB.
  4. If you issue a command >>mym in your MATLAB Command Window, and get a message "No connections open", then the setup was successful.

3.5 Example

Following example illustrates how to download Humber SHM Data by using shmYm which is used by humber_bridge_db class in the background. As shown in the example, the information you need to access Humber SHM Database is 1) DB Table Name (in this example, summary or gps Table), 2) DB Table Field Name (NOAA_WIS or GPH000EDH), 3) time window (tw1 or tw2) in MATLAB serial time format. That's it. Nothing else. 

%% USAGE EXAMPLE OF Humber SHM System
%

%% LOAD DATA
clear all;
do = humber_bridge_db; % Initialisation of humber_bridge_db class which uses shmYm in the background
tw1=datenum(2012,1,4,0,0,0)+[0 1]; % Time window of 1 day at 4th Jan 2012
% READ SUMMARY TABLE
r0=read(do,'summary',{'NOAA_WIS','NOAA_WID','GPH000EDH'},tw1); % a week record
tw2=datenum(2015,10,1,0,0,0)+[0 1/24/6];  % 10 min record just after midnight 1st Oct 2015
r1=read(do,'gps',{'GPH000WDE','GPH000WDN','GPH000WDH'},tw2);

%% PLOTTING
figure(1);
clf;
plot(r0.timestamp,r0.NOAA_WIS);
datetick('x','keeplimits');
ylabel('Windspeed (m/sec)');
title('NOAA\_WIS');

figure(2);
clf;
plot(r0.timestamp,r0.NOAA_WID);
datetick('x','keeplimits');
ylabel('Windspeed (Degree)');
title('NOAA\_WIS');

figure(3);
clf;
plot(r1.timestamp,r1.GPH000EDH);
datetick('x','keeplimits');
ylabel('Height (m)');
title('GPH000EDH');

Humber WIM system collects vehicle weight data. Details of the WIM system are not known for us. It is unlikely that HBB will be able to replace the faulty sensor until the A-frame project has ended, and calibration of the WIM system will not take place until that work has been done.

Access files are located at "c:\rms_data\2009_20XX_Humber_monitoring\03-Data\7-WIM" of FSDL-MON. A typical access file is as shown below (click the image to enlarge) and the file can be downloaded at the end of this page.

The access file contains numerous fields including Serial number (a unique number for each data record), Date, Time, Lane Number (1-4; 1 and 2 are for south-bound; 3 and 4 for north-bound), lane detail (north bound or south bound), direction (southbound, or northbound), direction (1=southbound, 2=northbound), Speed (km/h), speed_mph (mile/h), class_index (not sure about the meaning), class (not sure the meaning), length ( length of vehicle, not sure about the unit), axles (number of axles), gross (kg?; not sure about the unit), etc. However, important files for SHM application would be field as shown in the following table: Lane (Lane number 1-4), Direction_number (1=southbound, 2=northbound), speed (speed of vehicle km/h), speed_mph (speed of vehicle mile/h), class_index (class of vehicle, as mentioned not sure about the unit), class (not sure about the meaning), gross (gross weight of vehicle; unit seems kg; need to confirm). Most meaningful data field would be Gross (gross weight of vehicle) field.
